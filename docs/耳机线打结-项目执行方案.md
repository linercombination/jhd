# 耳机线在口袋中打结：路线1（SAW + 随机重排）项目执行方案（最终建模版，无 LaTeX）

研究目标（主问题A）：在固定口袋尺寸与扰动强度下，耳机线产生“打结”的概率 P_knot 如何随链长 L、刚度（持久长度代理 l_p）与端点状态（是否自由端）变化？

路线选择：统计模型 + 简化随机行走。将耳机线离散为三维自回避链（SAW），在“口袋盒子约束”内进行随机重排（pivot / crankshaft）采样；将“打结”定义为一个可计算的代理事件（本项目采用：存在环口即认为打结）。

---

## 0) 最终数学建模方案（你们报告的“方法”主干）

### 0.1 模型对象与假设

- 链：三维格点自回避链（SAW）。节点 r0…rN-1 属于 Z^3；相邻节点格点距离为 1；所有节点不重复（自回避）。
- 口袋：盒子约束，所有节点必须落在 [0,W]×[0,H]×[0,T] 内。
- 扰动：用 Monte Carlo 随机重排 moves（pivot 为主，可选 crankshaft）模拟“口袋内揉动/走动”。
- 刚度：用一个单调参数表征“更难弯折”。推荐使用最大弯折角阈值 θ_max（越小越硬）。

### 0.2 自变量与控制变量

自变量（要调控/分组对比）：
- 链长：N（等价于 L；取每段长度 b=1）。
- 刚度代理：θ_max（或 κ，但全项目只选一种实现，推荐 θ_max）。
- 端点状态：E ∈ {FF, AF, AA}（两端自由 / 一端固定 / 两端固定）。

控制变量（先固定，避免混淆；扩展时再放开）：
- 口袋尺寸 (W,H,T)：固定一套。
- 扰动强度：总步数 M=cN（固定 c，例如 500）。
- 环口判据参数：最小序号间隔 s（例如 10），检测间隔 τ（例如 10）。
- 每组重复次数：R（建议 ≥200；算力紧张可先 100）。

### 0.3 “打结”定义（代理：存在环口 = 打结）

对任一构型，若存在一对单体 (i,j) 满足：
- |i−j| ≥ s（避免把局部折返当“环”）
- dist(r_i, r_j) = 1（格点相邻；即出现“环口”）

则定义 knot_proxy = 1（本项目称“打结”）；否则 knot_proxy = 0。

单次仿真 run 的输出 Y 定义为：
- 在采样窗口内（M 步、每隔 τ 步检测一次），若任意时刻 knot_proxy=1，则 Y=1；否则 Y=0。

说明（写在报告里防质疑）：该定义是“可锁死缠结/环口事件”的代理，不是严格拓扑结；我们的结论解释为“在给定扰动强度下出现环口结构的概率随变量变化”。

### 0.4 概率估计与误差

对每个参数组 (N, θ_max, E) 独立重复 R 次，得到 Y_1…Y_R：
- 概率估计：P_hat = (1/R) * Σ Y_r
- 误差条（二项标准误）：SE = sqrt( P_hat * (1 − P_hat) / R )

输出即为你们最终要回答的问题：P_knot(N, 刚度, 端点状态)（在固定口袋与固定扰动强度下）。

### 0.5 统计建模（Logistic 回归，作为“数学建模”定量关系）

把每次 run 作为一条样本记录：
- 响应：Y（0/1）
- 自变量：log(N)、刚度标量 S、端点状态哑变量

推荐模型：
- logit(P) = β0 + β1*log(N) + β2*S + β3*I_AF + β4*I_AA + β5*(log(N)*S)

其中：
- P 为环口概率（即你们的 P_knot 代理）
- I_AF/I_AA 是端点状态的哑变量（FF 为基准）
- S 可取 cos(θ_max)（单调，且便于数值处理）或直接用 θ_max（注意单调方向即可）

汇报要点：β1 是否显著为正（更长更易“打结”）；刚度项是否显著降低概率；AF/AA 是否显著低于 FF；交互项是否显著（长且软更容易）。

## 2) 参数建议（最小可完成版本）

建议先按下面的“紧凑矩阵”跑通并出趋势，再考虑扩展。

## 3) 实现细节（按优先级：先跑通，再增强）

### 3.1 数据结构

- coords[N][3]：链坐标
- occupied：哈希集合（快速判重）
- rotations[24]：立方对称旋转（预生成）

### 3.2 SAW 初始化

- 推荐从盒子中部沿 x 方向直线放置；若超界则换方向或缩短再增长。
- 或用“增长-回退”生成：逐段随机选择可行方向，走不通就回退重试。

### 3.3 Pivot move（必做）

```text
pivot_step(coords):
  k ← Uniform{1..N-2}
  R ← Uniform(rotations)
  proposal ← copy(coords)
  for i = k+1..N-1:
    proposal[i] = coords[k] + R*(coords[i]-coords[k])
  if out_of_box(proposal): reject
  if has_duplicate_site(proposal): reject
  if stiffness_violate(proposal, theta_max): reject
  accept: coords = proposal
```

### 3.4 刚度检查（θ_max 硬约束）

- 对每个内点 i：
  - u = r_i - r_{i-1}
  - v = r_{i+1} - r_i
  - cosθ_i = u·v（格点上 |u|=|v|=1）
- 要求：cosθ_i ≥ cos(θ_max)

实现时建议直接比较 cos 值，避免反三角函数。

### 3.5 “环口=打结”检测（必做）

```text
knot_proxy(coords):
  for (i,j) with |i-j|>=s:
    if dist(coords[i], coords[j]) == 1:
      return 1
  return 0
```

### 3.6 单次 run 输出 Y（必做）

```text
run_once(N, theta_max, end_state, W,H,T, M, tau, s):
  init coords
  for step in 1..M_burn: do moves
  for step in 1..M:
    do moves
    if step % tau == 0:
      if knot_proxy(coords)==1:
        return 1
  return 0
```

## 4) 统计与图表（你们最终怎么“回答问题”）

### 4.1 概率与误差条

- 每组参数得到 P_hat 与 SE：
  - P_hat = mean(Y)
  - SE = sqrt(P_hat*(1-P_hat)/R)

### 4.2 Logistic 回归（推荐）

- 以每次 run 为样本，拟合：logit(P) = β0 + β1*logN + β2*S + β3*I_AF + β4*I_AA + β5*(logN*S)
- 输出：系数、p 值、置信区间；解释变量影响方向是否符合直觉。

### 4.3 必画的两张图

- P_hat vs N（每种端点状态一张图；图内用不同颜色表示刚度；带误差条）
- 热力图：横轴 N，纵轴刚度（θ_max），色值 P_hat；分别对 FF/AF/AA 做 3 张。

## 5) 风险点与报告写法（建议放“讨论”）

- 代理定义偏差：环口不等于严格拓扑结。报告里将 P_knot 明确为“环口出现概率”，并说明为何该结构与现实耳机线的锁死缠结相关。
- 采样相关性：连续 MC 步相关；用 burn-in 与间隔检测 τ 缓解。
- 格点粗粒化：强调研究重点是趋势和相对影响（N/刚度/端点状态）。
